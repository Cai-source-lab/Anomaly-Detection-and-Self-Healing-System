# WebAPP 健康巡检与自愈系统

一个基于 Python 的自动化运维监控工具，用于监控 Web 应用及基础设施的健康状态，并在发现故障时自动尝试修复，保障服务高可用。

## ✨ 功能特性

- ✅ **Web 服务健康检查**：HTTP 状态码检测、响应时间统计
- ✅ **端口存活检测**：TCP 端口连通性检查
- ✅ **进程状态监控**：进程存在性检查
- ✅ **系统资源监控**：CPU、内存、磁盘使用率监控
- ✅ **故障自动修复**：服务异常时自动重启，实现自愈
- ✅ **防雪崩设计**：重试次数限制，避免无限重启
- ✅ **结构化日志**：按天切割日志，便于排查问题
- ✅ **健康报告生成**：支持 Markdown/TXT 格式报告
- ✅ **配置与代码分离**：所有可配置项独立管理

## 🛠️ 技术栈

- **语言**：Python 3
- **核心库**：
  - `psutil` - 系统资源监控
  - `requests` - HTTP 请求
  - `PyYAML` - 配置文件解析
  - `logging` - 日志管理（标准库）
  - `socket` - 网络检查（标准库）

## 📁 项目结构
Health-Monitor-System
│
├── logger.py                 # 日志模块
│   └── 统一日志管理（控制台 + 文件 + 按天切割）
│
├── health_checker.py         # 健康检查模块
│   ├── Web服务检测（HTTP状态 / 响应时间）
│   ├── 端口检测（Socket连通性）
│   ├── 进程检测（psutil进程扫描）
│   └── 系统资源监控（CPU / 内存 / 磁盘）
│
├── auto_repair.py           # 自动修复模块
│   ├── 异常服务识别
│   ├── 多次重试机制（防雪崩）
│   └── 服务自动重启（subprocess）
│
├── config.yaml              # 配置中心
│   ├── 监控服务列表
│   ├── 阈值配置（CPU/内存/磁盘）
│   └── 重启命令映射
│
├── monitor.py               # 主调度入口（巡检流程编排）
│   └── 串联 检查 → 日志 → 修复 → 报告
│
├── logs/                    # 日志输出目录
│   ├── monitor.log
│   └── 轮转日志文件
│
└── reports/                 # 报告输出目录
    └── health_report_xxx.md
